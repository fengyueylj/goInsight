#### 提交流程
```bash
-> 开发（具有提交工单权限）

  -> 提交DDL或DML工单

    -> 语法检测

      -> 符合语法规则 -> 提交

      -> 不符合语法规则（goInception返回） -> 提示开发修改 -> 继续语法检测

  -> 系统发送消息通知相关人员（提交人、审核人、抄送人、复核人）需要启用钉钉、微信或邮件（config.py配置）

  -> 审核人审核（具有超级权限的用户可以一键通过审核，比如后台的admin用户，后台勾选用户详情里面的【超级用户状态】即可）
     - 拥有审核权限且您为当前工单的审核人

  -> 审核通过后，DBA执行或有执行权限的用户执行

  -> 执行完成后，提交人检查数据是否复核预期

  -> 提交人检查完成后，点击【复核】按钮将工单状态改为：已复核

  -> 提交人此时可以点击【钩子】按钮，将工单勾到下一个将要执行的环境
```

#### 钩子流程
```bash
工单 -> 提交到 -> 测试环境 

    测试环境执行完成 -> 钩子 -> 预发布环境

       预发布环境执行完成 -> 钩子 -> 生产环境
```
